也是对搭建博客的过程的一个复习和深入了解吧，也可以给小白做参考。

准备前置工具

安装：

  1. Node.js ( 会附带 npm )

    在官网下载安装即可。node.js 是 hexo 前置软件，npm 则是 node.js 其中附带的包管理器。如果一切顺利，那么，在控制台（ cmd 和 PowerShell ）运行node -v和npm -v应当可以看到 node.js 和 npm 的版本号。

  2. Git

    官网安装即可。安装完成后，可以启用Git Bash。

  3. cnpm 和 hexo 部署插件

    hexo 是基于 node.js 的博客框架，本来应该是使用 npm 进行安装，不过，考虑到连接外网速度的问题，可以先换一个下载源。

    在命令行界面下，键入npm install -g cnpm --registry=https://registry.npm.taobao.org，即可安装 cnpm 。（相当于先下了一个高速下载器来下文件）。

    cnpm 安装完成，于是键入 cnpm install -g hexo-cli 即可安装 hexo 了（注意和上一个命令的相似性)。如果一切顺利，应该可以运行hexo -v 校验版本安装成功与否和校验版本。

正式的步骤

初始化一个博客

新建一个文件夹 ( 这个文件夹会在本地存储博客的所有内容 )。所以，要在喜欢的地方新建并命名成喜欢且易记忆的名字。

可以一开始就使用命令行进行操作，也可以用更常见的方法新建，然后在文件夹内右键选择 Git Bash Here 。无论如何，在该目录下，Git Bash 中输入hexo init即可（可以加上sudo以使用管理员权限）

启用博客

继续，在上述的博客文件夹下运行 Git Bash 。输入hexo s即可开始（start）编译，并在 localhost:4000 这个内部地址上展示。如果一切正常，那么就应该看到一篇名为 Hello World 的博文在博客网页上欢迎我们的到来。

推送到远端（GitHub）

用别的其实也ok，比如说 coding 。因为GitHub是真的慢

首先需要一个 GitHub 账号。

然后，点选 Repository 边的 New 新建一个库。记住 repository name 只能是 “ GitHub 用户名.github.io” 的形式。

安装 Git 部署插件：回到 Git Bash 界面，在博客目录下输入cnmp install --save hexo-deployer-git安装 Git 部署插件以把博客部署到 Github 上（注意 Git 和 GitHub 不是一回事哦）。

修改配置文件：在博客目录下找到 _config.yml 用编辑器打开（记事本也行） 。滑到最低端找到 deploy 。把 type 后面引号中写入 git 。然后换行写上：repo： ……把省略号换成在 GitHub 建立的库的网址。再换行写上branch: master 。记住是英文冒号加空格。

部署：用hexo g来生成（generate）HTML网页文件，键入git config --global user.email "xxx"和git config --global user.name "xxx"配置 GitＨub 账号的邮件地址和用户名（替换 xxx ）。然后键入 hexo d 来部署（deploy）到远端。第一次部署应当会出现弹出窗口要求输入密码。总之，一步一步按要求键入吧，没有选择的。

然后，打开浏览器，网址栏输入建立库的地址．就能看到个人博客了．All Done !
